# 문서 경계 규칙

## 목적

rule 문서, guide 문서, implementation 기록 문서, OMX runtime state의 차이를 정의한다.

## 디렉토리 역할

- `README.md`: 저장소 루트의 대표 사람이 읽는 요약 문서
- `rule/`: 기준 Codex 실행 규칙
- `docs/guide/`: 사람이 실제로 따라야 하는 사용자 가이드
- `docs/implementation/`: 사람이 읽는 최종 브리핑과 결과 문서
- `.omx/`: 내부 coordination을 위한 OMX runtime state이며 사용자-facing 문서 영역이 아니다.
