# Code Implementation

Read [`rule/index.md`](../../../rule/index.md), [`rule/rules/instruction-model.md`](../../../rule/rules/instruction-model.md), [`rule/rules/subagent-orchestration.md`](../../../rule/rules/subagent-orchestration.md), [`rule/rules/cycle-document-contract.md`](../../../rule/rules/cycle-document-contract.md), and [`rule/rules/language-policy.md`](../../../rule/rules/language-policy.md) before editing.

Use this skill for implementation work.

- Apply approved change plans to code and configuration.
- Keep the change set focused and pair it with targeted verification.
- Keep user-facing docs separate from working records.
- Do not redefine the plan; leave implementation rationale as references to the relevant rules and cycle document.
