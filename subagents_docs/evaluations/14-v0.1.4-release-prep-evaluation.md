## 주요 판정 근거

### 1. 최신 공개 릴리스 표기가 `v0.1.4`로 갱신됐다

- `README.md:35`는 `The current latest public release is \`v0.1.4\`.`로 표시된다.
- `README.ko.md:35`는 `현재 최신 공개 릴리스는 \`v0.1.4\`입니다.`로 표시된다.

### 2. 이전 `v0.1.3` 최신 릴리스 표기는 해당 위치에 남아 있지 않다

- `README.md`, `README.ko.md`를 대상으로 `latest public release|최신 공개 릴리스|v0\.1\.3|v0\.1\.4` 검색을 수행했다.
- 검색 결과는 두 README의 `v0.1.4` 라인만 반환됐고, 최신 공개 릴리스 문맥의 `v0.1.3` 잔존은 없었다.

### 3. 이번 릴리스 준비 변경은 계획된 최소 범위를 유지했다

- 계획 문서는 수정 대상을 `README.md`, `README.ko.md`로 제한했다.
- 구현 기록도 같은 두 파일만 수정했다고 보고한다.
- 실제 릴리스 준비 변경의 목적은 최신 공개 릴리스 표기 갱신뿐이며, `hs-init-project/`는 이번 릴리스 준비 단계에서 추가 변경 대상으로 삼지 않았다.

### 4. 이 준비 단계 기준으로 `v0.1.4` 커밋/태그/푸시를 진행할 수 있다

- README의 최신 공개 릴리스 표기는 일관되게 `v0.1.4`로 맞춰졌다.
- 공개 태그는 현재 `v0.1.3`까지 존재하므로 `v0.1.4`는 다음 patch 태그로 적절하다.
- 따라서 이번 준비 단계 기준으로는 commit / push / tag / tag push를 진행할 수 있는 상태다.
